商家模块管理清单（基于需求文档当前版本）

说明：
- 本文档聚焦商家角色（role=2）
- 目标：先完成“轻资料 + 浏览筛选 + 直接下单”闭环
- 不走“标准需求单/发布需求”模式

一、商家定位（按需求文档）
- 商家核心路径：注册（锁定经纪人） -> 浏览模特 -> 筛选 -> 下单 -> 支付 -> 确认完成 -> 评价
- 平台当前阶段：小程序端为主，线上下单、线下履约

二、商家资料字段（最小可用）
1) 必填字段（当前建议）
- role=2（商家）
- phone（手机号，微信授权绑定）
- city（所在城市，用于“只接本地单”匹配）

2) 选填字段（建议）
- nickname（昵称）
- avatar_url（头像）
- company_name（公司名，企业商家可填）
- contact_name（联系人）
- contact_phone（联系电话）

3) 条件字段（规则触发）
- verified_status（实名认证状态）
- 规则：商家实名认证可选；当订单金额超过阈值（如5000）时要求实名

三、商家流程（MVP）
1) 登录注册
- 微信登录 -> 游客
- 选择身份为商家 -> 手机授权 -> 完成注册（role=2）

2) 浏览筛选
- 按分类、风格、场景、城市、价格、档期、评分等过滤
- 列表进入模特详情，查看资料与价格、可接单日期

3) 下单支付
- 选择模特、日期、时长、需求描述
- 系统计算总价（模特服务费 + 平台服务费）
- 支付后进入订单状态机

4) 履约确认
- 模特接受/拒绝
- 服务完成后商家确认
- 确认完成触发分账

5) 评价
- 订单完成后评分 + 文本 + 图片（按文档约束）

四、与模特侧联动规则
- 仅展示 `is_available=1` 的模特
- 仅允许下单日期为档期 `available`
- 订单确认后（后续）自动将对应日期标记为 `full`
- 若模特设置“只接本地单”，商家城市需与模特城市匹配

五、当前实现状态（截至当前）
1) 已实现
- 商家可通过注册流程选择 role=2 并完成注册
- 登录/JWT/手机号绑定/完成注册链路打通
- 模特侧资料、价格、档期、接单偏好基础能力已可用（为商家筛选提供数据基础）

2) 未完成
- 商家专属资料页（city/company/contact）完善
- 商家端筛选页面与详情页面（完整）
- 商家下单与支付完整流程
- 订单状态机全链路落库与驱动
- 商家评价能力

六、数据库建议（商家侧）
- 现有 `merchant_profiles` 可作为商家扩展主表：
  - company_name
  - contact_name
  - contact_phone
  - city
- 建议补充（后续）：
  - verified_status（若需商家独立实名状态可加，或沿用 users.verified_status）
  - business_license_no（企业认证扩展）

七、接口建议（商家侧最小集）
- GET /api/merchant/me
- PUT /api/merchant/profile
- GET /api/models/search
- GET /api/models/:id
- POST /api/orders
- POST /api/orders/:id/pay
- POST /api/orders/:id/confirm
- POST /api/orders/:id/review

八、近期优先级（建议）
P0
- 商家资料最小字段页（city 必填）
- 模特搜索筛选接口（先支持分类/城市/价格/档期/接单开关）
- 商家下单接口（校验档期可接单）

P1
- 支付回调 + 订单状态机
- 商家确认完成 + 评价
- 高额订单实名认证门槛

P2
- 商家看板与复购能力（收藏、常用模特）
- 智能推荐与排序优化
