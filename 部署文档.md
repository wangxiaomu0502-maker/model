# 星链项目部署文档（前后端）

## 1. 服务器与域名

- 服务器 IP：`118.195.130.77`
- 管理端域名：`admin.xinglianmoku.cn`
- 接口域名：`api.xinglianmoku.cn`

DNS 建议：

- `admin` -> `A` -> `118.195.130.77`
- `api` -> `A` -> `118.195.130.77`

## 2. 服务器运行结构

- 前端静态目录：`/var/www/admin.xinglianmoku.cn`
- 后端目录：`/srv/xinglian-server`
- Nginx 配置目录：`/etc/nginx/conf.d`
- 证书目录：`/etc/nginx/ssl`
- 后端进程管理：`pm2`（进程名：`xinglian-server`）

## 3. 前端部署（管理端）

### 3.1 本地打包

在 `xinglian-admin` 目录执行：

```bash
VITE_API_BASE_URL="https://api.xinglianmoku.cn" npm run build
```

### 3.2 上传到服务器

```bash
scp -r dist/. root@118.195.130.77:/var/www/admin.xinglianmoku.cn/
```

### 3.3 验证

```bash
curl -I https://admin.xinglianmoku.cn/
```

应返回 `HTTP/2 200`。

## 4. 后端部署（API）

### 4.1 本地构建

在 `xinglian-server` 目录执行：

```bash
npm run build
```

### 4.2 上传代码（不含 node_modules/dist）

```bash
tar --exclude="node_modules" --exclude="dist" -czf /tmp/xinglian-server.tar.gz .
scp /tmp/xinglian-server.tar.gz root@118.195.130.77:/srv/xinglian-server/
scp .env root@118.195.130.77:/srv/xinglian-server/.env
```

### 4.3 服务器重启服务

```bash
ssh root@118.195.130.77
cd /srv/xinglian-server
tar -xzf xinglian-server.tar.gz && rm -f xinglian-server.tar.gz
npm ci
npm run build
pm2 restart xinglian-server || pm2 start dist/index.js --name xinglian-server
pm2 save
```

### 4.4 验证

```bash
curl -I https://api.xinglianmoku.cn/api/admin/login
```

`GET` 该登录接口通常会返回 `404`（路由/方法不匹配），但说明域名和反代已通。

## 5. Nginx 配置

### 5.1 admin 配置：`/etc/nginx/conf.d/admin.xinglianmoku.cn.conf`

```nginx
server {
    listen 80;
    server_name admin.xinglianmoku.cn;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name admin.xinglianmoku.cn;

    ssl_certificate /etc/nginx/ssl/admin.xinglianmoku.cn_nginx/admin.xinglianmoku.cn_bundle.crt;
    ssl_certificate_key /etc/nginx/ssl/admin.xinglianmoku.cn_nginx/admin.xinglianmoku.cn.key;

    root /var/www/admin.xinglianmoku.cn;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|svg|ico|woff|woff2)$ {
        expires 7d;
        add_header Cache-Control "public, max-age=604800";
    }
}
```

### 5.2 api 配置：`/etc/nginx/conf.d/api.xinglianmoku.cn.conf`

```nginx
server {
    listen 80;
    server_name api.xinglianmoku.cn;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.xinglianmoku.cn;

    ssl_certificate /etc/nginx/ssl/api.xinglianmoku.cn_nginx/api.xinglianmoku.cn_bundle.crt;
    ssl_certificate_key /etc/nginx/ssl/api.xinglianmoku.cn_nginx/api.xinglianmoku.cn.key;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

### 5.3 gzip 配置：`/etc/nginx/conf.d/gzip.conf`

```nginx
gzip on;
gzip_comp_level 6;
gzip_min_length 1024;
gzip_vary on;
gzip_proxied any;
gzip_types text/plain text/css text/xml text/javascript application/javascript application/x-javascript application/json application/xml application/xml+rss image/svg+xml font/ttf font/otf application/vnd.ms-fontobject application/font-woff application/font-woff2;
```

### 5.4 重载

```bash
nginx -t && systemctl reload nginx
```

## 7. 常见问题

- 前端仍请求旧地址：浏览器强刷（`Cmd+Shift+R`）或禁用缓存后刷新。
- `https` 不通：检查证书路径、443 监听、防火墙/安全组。
- API 域名返回 404：先确认请求方法（登录接口应为 `POST`）。
- 部署后白屏：检查 `admin` 的 `index.html` 是否引用最新 `assets/*.js`。

## 8. 一键发布脚本

根目录已提供脚本：`deploy.sh`  
作用：一键完成前端构建+上传、后端构建+上传、PM2 重启、线上连通性检查。

### 8.1 依赖

- 本地需安装 `sshpass`

### 8.2 用法

```bash
cd /Users/edz/Documents/code/模特
chmod +x deploy.sh
./deploy.sh --host 118.195.130.77 --user root --pass '你的服务器密码'
```

### 8.3 脚本行为

- 前端使用 `https://api.xinglianmoku.cn` 作为 API 基地址构建
- 上传前端 `dist` 到 `/var/www/admin.xinglianmoku.cn`
- 上传后端源码包和 `.env` 到 `/srv/xinglian-server`
- 服务器端构建后端并 `pm2 restart xinglian-server`
- 自动输出 `admin/api` 的 HTTP 头用于验证
